# 模型主要模块说明

本文档说明当前项目主线模型中各个主要模块的功能、输入输出以及它们在整体训练流程中的作用。当前主线结构可以概括为：

`数据预处理 -> MacBERT Encoder -> TSBECL 边界分支 -> GlobalPointer 主分支 -> MCN 对齐损失 -> FGM 对抗训练 -> 实体解码与评估`

---

## 1. 数据预处理模块

### 1.1 原始作用

数据预处理模块负责把原始标注样本整理成模型可以直接使用的张量输入，同时保证字符级实体区间能够正确映射到 token 级区间。

### 1.2 主要操作

1. 读取原始文本和实体标注。
2. 用 tokenizer 对文本编码，得到：
   - `input_ids`
   - `token_type_ids`
   - `attention_mask`
   - `offset_mapping`
3. 根据 `offset_mapping` 构建：
   - `start_mapping`
   - `end_mapping`
4. 把字符级实体 `(start_char, end_char, label_id)` 映射到 token 级实体 `(start_token, end_token, label_id)`。
5. 生成四维 span 标签矩阵：
   - 形状为 `batch_size x entity_type_size x seq_len x seq_len`
6. 对 batch 内不同长度样本做 padding。

### 1.3 作用

- 把原始标注转成 GlobalPointer 可直接训练的 span 监督信号。
- 保证训练和评估都使用统一的数据对齐口径。
- 为后续实体级回译提供 `offset_mapping`。

---

## 2. 同类实体替换增强模块

### 2.1 原始作用

该模块在训练阶段对样本做轻量增强，用同 batch 内相同类别的其他实体文本替换原句中的某个实体片段，从而增加模型对实体表面形式变化的鲁棒性。

### 2.2 主要操作

1. 在当前 batch 中收集每个实体类别对应的实体文本池。
2. 随机挑选一个可替换实体。
3. 用同类实体池中的另一个实体文本替换它。
4. 同步修正替换后文本中的实体偏移位置。

### 2.3 作用

- 缓解模型对少数固定实体词面的过拟合。
- 增强实体上下文变化下的泛化能力。
- 对 recall 往往有帮助，但增强过强时可能带来一定误报。

---

## 3. MacBERT Encoder 模块

### 3.1 原始作用

MacBERT 是整个模型的语义编码主干，用来把输入 token 序列映射成上下文相关的连续向量表示。

### 3.2 输入输出

输入：

- `input_ids`
- `attention_mask`
- `token_type_ids`

输出：

- `encoder_hidden`

其中 `encoder_hidden` 的形状通常为：

`batch_size x seq_len x hidden_size`

### 3.3 作用

- 提供句子级上下文语义表示。
- 为边界分支和 GlobalPointer 主分支提供共同的基础表示。
- 决定模型对实体语义和上下文依赖的建模能力上限。

---

## 4. TSBECL 边界分支

### 4.1 原始作用

TSBECL 边界分支专门建模“哪些 token 更像实体起点、哪些 token 更像实体终点”，用显式边界信号增强 span 主任务。

### 4.2 内部子模块

#### 4.2.1 Head GRU

对 `encoder_hidden` 做一条双向 GRU 编码，重点学习偏向实体起点的信息。

输出：

- `head_hidden`

#### 4.2.2 Tail GRU

对 `encoder_hidden` 做另一条双向 GRU 编码，重点学习偏向实体终点的信息。

输出：

- `tail_hidden`

#### 4.2.3 起止点分类器

分别对 `head_hidden` 和 `tail_hidden` 做线性分类，得到：

- `start_logits`
- `end_logits`

它们用于监督每个 token 是否为实体起点或终点。

#### 4.2.4 边界增强融合

将：

- `head_hidden`
- `tail_hidden`
- `encoder_hidden`

通过线性投影融合得到：

- `fused_hidden`

同时还保留：

- `boundary_hidden`

其中当前实现里，`boundary_hidden` 由 `head_hidden` 和 `tail_hidden` 平均得到，用于 MCN 对齐模块。

### 4.3 作用

- 显式建模边界信息。
- 给 GlobalPointer 提供边界增强后的 token 表示。
- 提升模型对边界定位的敏感性。
- 为后续 MCN 对齐提供边界视角的 token 表示。

---

## 5. GlobalPointer 主分支

### 5.1 原始作用

GlobalPointer 是当前模型的主预测头，负责对所有可能的 `(实体类型, 起点, 终点)` span 进行统一打分。

### 5.2 内部子模块

#### 5.2.1 Dense 投影

将 `fused_hidden` 投影成两套表示：

- `query_states`
- `key_states`

每种实体类型都拥有对应的 query/key 子空间。

#### 5.2.2 RoPE 位置编码

对 `query_states` 和 `key_states` 应用旋转位置编码，使模型能够感知 span 的相对位置关系。

#### 5.2.3 Span 打分

用点积方式计算每个类型下任意起止 token 对的分数，得到：

- `span logits`

形状为：

`batch_size x entity_type_size x seq_len x seq_len`

#### 5.2.4 Mask 约束

对 `span logits` 做两类约束：

1. `pad mask`：屏蔽 padding 位置
2. 下三角 mask：只保留 `start <= end` 的 span

### 5.3 作用

- 直接完成实体 span 的主任务预测。
- 允许天然处理嵌套实体和重叠实体。
- 提供实体级评估和解码所需的核心打分矩阵。

---

## 6. MCN 对齐模块

### 6.1 原始作用

MCN 对齐模块借鉴 `mcn_model.py` 中的 co-energy 思想，不直接照搬视觉检测结构，而是把“跨分支注意力对齐”迁移到 NER 中。

这里它的目标是：

让 **边界分支关注的 token 区域** 和 **GlobalPointer 主任务认为重要的 token 区域** 在统一的 token 相似空间中尽量一致。

### 6.2 输入

当前设计中，MCN 对齐模块使用三类信息：

1. `boundary_hidden`
   - 来自边界分支
   - 表示边界视角下的 token 表示

2. `encoder_hidden`
   - 来自 MacBERT Encoder 原始输出
   - 表示基础上下文语义

3. `span logits`
   - 来自主分支
   - 表示模型主任务认为哪些 span 更像实体

同时还使用：

- `start_logits`
- `end_logits`
- `attention_mask`

### 6.3 内部子模块

#### 6.3.1 boundary_attention

由 `start_logits + end_logits` 生成 token 级注意力分布，表示：

“边界分支当前认为哪些 token 更重要”

#### 6.3.2 span_attention

从四维 `span logits` 反推出 token 级分布，表示：

“GlobalPointer 主分支当前认为哪些 token 更属于实体区域”

#### 6.3.3 token_similarity

对：

- `boundary_hidden`
- `encoder_hidden`

做归一化后计算 token 两两之间的相似矩阵。

它刻画的是：

“边界视角下的重要 token 与基础语义视角下的重要 token 是否一致”

#### 6.3.4 co-energy

用如下思想聚合：

`boundary_attention -> token_similarity -> span_attention`

得到一个 co-energy 分数。

如果边界关注区域和 span 关注区域在语义空间内一致，这个分数就更高。

### 6.4 作用

- 让边界分支与主任务分支形成更稳定的一致性约束。
- 减少边界头和 span 头各说各话的问题。
- 强化边界关注与实体区域关注之间的对齐关系。

---

## 7. 损失函数模块

### 7.1 span_loss

作用：

监督 GlobalPointer 主分支学习正确的实体 span。

特点：

- 使用多标签形式训练
- 一个样本中允许存在多个实体
- 允许嵌套和重叠实体

### 7.2 boundary_loss

作用：

监督：

- `start_logits`
- `end_logits`

使模型显式学习实体起点和终点。

### 7.3 mcn_alignment_loss

作用：

通过 co-energy 对齐约束边界分支和主任务分支的 token 注意力分布。

### 7.4 总损失

总损失由三部分组成：

`total_loss = span_loss + boundary_loss + mcn_alignment_loss`

作用：

- `span_loss` 保证实体主任务性能
- `boundary_loss` 强化边界定位
- `mcn_alignment_loss` 强化跨分支一致性

---

## 8. FGM 对抗训练模块

### 8.1 原始作用

FGM 在 embedding 层加入一次微小扰动，再重新前向和反向传播，用于提升模型对输入扰动的鲁棒性。

### 8.2 主要操作

1. 正常前向计算损失。
2. 反向传播得到梯度。
3. 在 embedding 参数上沿梯度方向施加扰动。
4. 再做一次前向和反向。
5. 恢复原始参数。

### 8.3 作用

- 提高泛化能力。
- 降低模型对局部输入噪声的敏感度。
- 在医疗实体识别这种高相似语义场景中，常常能带来更稳的验证表现。

---

## 9. 解码与评估模块

### 9.1 解码

从 `span logits` 中取出满足阈值条件的 span：

1. 找出 `score > prediction_threshold` 的位置
2. 将 token 级 span 回映射为字符级 span
3. 生成实体结果

### 9.2 评估指标

当前主要输出：

- `overall F1`
- `boundary F1`
- `rare F1`
- `long F1`
- `per_type F1`

### 9.3 作用

- `overall F1`：衡量总体实体识别能力
- `boundary F1`：衡量边界定位能力
- `rare F1`：衡量稀有实体识别能力
- `long F1`：衡量长实体识别能力
- `per_type F1`：观察不同类别的优劣势

---

## 10. 当前主线的整体理解

当前这条主线不是单纯的 GlobalPointer，也不是单纯的边界头模型，而是一个“主任务 + 边界增强 + 分支对齐 + 对抗训练”的组合：

1. MacBERT 提供基础语义表示。
2. TSBECL 边界分支负责学习边界信息。
3. GlobalPointer 负责统一 span 打分。
4. MCN 对齐模块负责把边界分支和主任务分支拉到更一致的注意力空间里。
5. FGM 负责进一步提升训练鲁棒性。

因此，这条主线的核心思想可以概括为：

**用边界建模增强 span 主任务，再用跨分支对齐约束提升两者的一致性。**
